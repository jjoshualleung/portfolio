The Story Behind the Material Tag Generator

Scenario & Motivation

In a warehouse environment, material tags are essential for tracking and organizing components for rack servers, switches, and other hardware. Previously, these tags were generated using an Excel macro developed six years ago. However, when the developer left, the macro became outdated and riddled with bugs, leading to inefficiencies and frustrations among warehouse operators.

The macro required manual input and could only generate tags one column at a time. A single Excel file could contain up to 26 columns, taking operators 1-2 hours to process. Recognizing the inefficiency and its impact on productivity, I decided to create a modern, automated solution.

The Problem

The outdated Excel macro posed several challenges:

  • Frequent bugs and lack of support due to the original developer's resignation.
  • Inability to process multiple columns or files simultaneously, resulting in time-consuming, manual work.
  • Prone to human error during manual input and processing.

Proposed Solution

I envisioned a web application that could:

  • Automatically generate material tags for entire Excel files, regardless of the number of columns.
  • Process multiple Excel files simultaneously with a single click.
  • Provide a user-friendly interface that eliminates manual input and ensures data accuracy.

My goal was to replace the Excel macro with a robust, scalable, and self-contained web application, significantly reducing the time and effort required to generate material tags.

The Process

I followed a structured approach to build and deploy the application:

  1. Building the Functionality: I used Python and the OpenPyXL library to develop the core functionality for generating material tags. I validated the functionality with unit tests using Pytest to ensure reliability and accuracy.
  2. Designing the Web Application: I created a user-friendly interface using HTML, Jinja, and CSS. The web application allows users to upload multiple Excel files and generates all material tags in less than a second.
  3. Deploying the Application: I containerized the application using Podman and configured it with a Gunicorn server and a Dockerfile for self-containment. For deployment, I set up a Linux Ubuntu server, ensuring the application was accessible and performant in a production environment.
