Project Overview

adjusttxt is a Java-based command-line and library utility developed using an agile, test-driven approach. It performs basic line-based text transformations such as skipping, trimming, prefixing, blank-line removal, and reversing. The tool was designed to evolve through rigorous TDD and interface separation for reuse beyond the CLI.

Development Workflow

  • TSL Creation: Generated a category-partition Test Specification List to cover all input scenarios.
  • JUnit Testing: Built a JUnit regression suite based on the TSL.
  • CLI Implementation: Implemented the CLI based on the parser and logic layers.
  • Interface Refactoring: Refactored the functionality into a reusable class implementing adjusttxtInterface with typed exceptions.

Core Features and Options

  • -s N: Skip the first N lines
  • -t: Trim whitespace on each line
  • -b: Delete blank lines
  • -p "str": Prefix each line with a string
  • -w: Reverse words in each line
  • -x: Reverse the order of all lines

Architecture Snapshot

The application is split into modular components for parsing, transformation, and output.

Validation & Outcomes

  • Developed 62 test cases from a category-partition TSL to cover a wide range of inputs and edge cases.
  • Achieved 90% pass rate on the instructor’s hidden autograder test suite via Gradescope CI.
  • Validated behavior across:
    – Conflicting or missing flags (-w vs -x, invalid skip count)
    – Edge cases like blank files, no trailing newline, or single-line inputs
    – Flag order independence and trimming vs prefix interaction
